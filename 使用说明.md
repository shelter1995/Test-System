# AI话术陪练系统 - 使用说明

## 系统概述

AI话术陪练系统是一个基于 MiniMax AI 和统一传统 RAG 的智能角色扮演训练系统。系统能够：
- 基于知识库内容进行智能对话
- 提供多种角色扮演场景
- 生成专业培训材料和解决方案
- 使用传统 RAG 快速处理常见文档
- 通过 OCR 与转写能力处理复杂 PDF、图片、音频、视频等内容

## 系统架构

系统包含两个核心服务：

1. **知识库服务**：端口 8003，统一使用传统 RAG 进行导入、解析与检索。
2. **AI 话术陪练系统**：端口 8002，负责场景会话、MiniMax 对话生成、实时评估和报告。

## 快速开始

### 启动服务

推荐在项目根目录运行一键启动脚本：

```bat
start_services.bat
```

也可以手动启动两个服务：

1. 启动 RAG 核心服务：
   ```bash
   cd rag-anything-api
   python start.py
   ```

2. 启动 AI 陪练系统：
   ```bash
   cd ai-tutor-system
   python tutor_backend.py
   ```

3. 打开统一工作台：
   - 浏览器访问：http://localhost:8002

### 统一工作台

启动 `rag-anything-api` 和 `ai-tutor-system` 后，访问：

- 工作台：http://localhost:8002
- RAG API 文档：http://localhost:8003/docs
- 陪练/生成 API 文档：http://localhost:8002/docs

工作台包含：

1. **总览**：查看服务状态、知识库数量、文件数量和最近产物。
2. **知识库**：创建产品知识库，引入文件并查看导入日志。
3. **知识库问答**：切换知识库，多轮提问，并查看来源依据。
4. **内容生成**：选择知识库后生成解决方案、培训讲义、测试题和 README。
5. **陪练系统**：选择知识库、客户场景和产品后开始对话陪练。
6. **模型设置**：配置推理模型、嵌入模型和重排模型。
7. **历史产物**：查看并下载生成的 Markdown 文件。

## RAG核心服务使用

### 1. 知识库管理

#### 导入文档/目录
```bash
# 导入单个文件
curl -X POST "http://localhost:8003/ingest/path" \
  -H "Content-Type: application/json" \
  -d '{"path": "D:/GitHub_WorkSpace/Test-System/商务彩铃/商务彩铃_OCR.txt", "database": "商务彩铃"}'

# 导入目录（递归）
curl -X POST "http://localhost:8003/ingest/path" \
  -H "Content-Type: application/json" \
  -d '{"path": "D:/GitHub_WorkSpace/Test-System/商务彩铃", "database": "商务彩铃", "recursive": true}'
```

#### 查询文档
```bash
# 使用API查询文档
curl -X POST "http://localhost:8003/search" \
  -H "Content-Type: application/json" \
  -d '{"query": "商务视频彩铃业务办理流程", "database": "商务彩铃", "n_results": 5}'
```

### 2. API接口说明

#### 文件/目录导入
- **端点**: `/ingest/path`
- **方法**: `POST`
- **参数**:
  - `path`: 文件或目录路径
  - `database`: 知识库ID（例如：`商务彩铃`）
  - `recursive`: 目录是否递归（可选）

#### 文档搜索
- **端点**: `/search`
- **方法**: `POST`
- **参数**:
  - `query`: 查询文本
  - `database`: 知识库ID（例如：`商务彩铃`）
  - `n_results`: 返回结果数量（默认：10）

#### 轻量上下文检索

- **端点**: `/context`
- **方法**: `POST`
- **用途**: 实时陪练优先使用，返回知识上下文，避免每次检索都生成完整答案。
- **参数**: `query`（查询文本）、`database`（知识库ID）、`n_results`（结果数量）

#### 系统状态
- **端点**: `/status`
- **方法**: `GET`
- **返回**: 系统运行状态和数据库统计信息

### 3. 数据库管理

#### 查看数据库统计
```bash
curl -s http://localhost:8003/db/stats/商务彩铃
```

#### 重新处理文件
```bash
curl -X POST "http://localhost:8003/db/商务彩铃/documents/{sha256}/retry" \
  -H "Content-Type: application/json" \
  -d '{"strategy":"markdown_segments"}'
```

前端文档列表也提供「重试」按钮。点击后按钮会进入处理中状态，并在上传日志中显示开始、成功或失败信息。

## AI话术陪练系统使用

### 1. 基本操作

#### 选择场景
1. 打开前端界面
2. 从下拉菜单中选择角色扮演场景
3. 点击"开始对话"按钮

#### 对话操作
- 在输入框中输入你的问题或回答
- 点击"发送"按钮或按Enter键发送
- 系统会基于知识库内容生成回复

#### 保存会话
- 系统会自动保存会话到 `ai-tutor-system/tutor_data/sessions/` 目录
- 会话文件命名格式：`session_YYYYMMDD_HHMMSS.json`

### 2. 技能系统

系统支持两种专业技能：

#### 培训生成技能 (peixun-skill)
**功能**: 基于成人学习理论生成专业培训材料

**使用方法**:
1. 在对话中输入：`使用培训生成技能`
2. 系统会引导你输入培训主题和目标受众
3. 系统生成结构化的培训材料，包括：
   - 培训目标
   - 课程大纲
   - 教学方法
   - 评估方式
   - 学习资源

**示例**:
```
用户: 使用培训生成技能
系统: 请输入培训主题：
用户: 商务视频彩铃销售技巧
系统: 请输入目标受众：
用户: 客户经理
系统: [生成培训材料]
```

#### 解决方案生成技能 (solution-generator-skill)
**功能**: 基于SCQA MECE架构生成个性化解决方案

**使用方法**:
1. 在对话中输入：`使用解决方案生成技能`
2. 系统会引导你输入客户信息和需求
3. 系统生成结构化的解决方案，包括：
   - 现状分析
   - 问题界定
   - 解决方案
   - 实施计划
   - 预期效果

**示例**:
```
用户: 使用解决方案生成技能
系统: 请输入客户名称：
用户: 阳光电源股份有限公司
系统: 请描述客户需求：
用户: 需要提升企业形象和品牌知名度
系统: [生成解决方案]
```

### 3. 场景配置

系统支持多种角色扮演场景，配置文件位于：`ai-tutor-system/tutor_config.py`

**场景示例**:
- 商务视频彩铃销售场景
- 客户服务场景
- 产品演示场景
- 技术支持场景

**自定义场景**:
编辑 `tutor_config.py` 文件，添加新的场景配置：
```python
"新场景名称": {
    "description": "场景描述",
    "system_prompt": "系统提示词",
    "initial_message": "初始消息"
}
```

## 高级功能

### 1. 知识库解析能力

用户可见的知识库能力统一为传统 RAG（向量检索 + 重排），不再提供 RAG-Anything 引擎切换入口。

解析依赖：

- `MinerU`：扫描 PDF、复杂 PDF、图片 OCR 解析
- `LibreOffice`：`.doc/.xls/.ppt` 老 Office 格式转换
- `ffmpeg`：视频抽取音轨
- `openai-whisper`：音频/视频语音转写

支持格式：

- 文档：`.pdf`、`.doc`、`.docx`、`.xls`、`.xlsx`、`.ppt`、`.pptx`、`.txt`、`.md`、`.csv`
- 图片：`.png`、`.jpg`、`.jpeg`、`.bmp`、`.tiff`、`.webp`
- 音频：`.mp3`、`.wav`、`.flac`、`.aac`、`.ogg`、`.m4a`、`.wma`
- 视频：`.mp4`、`.avi`、`.mkv`、`.mov`、`.webm`、`.wmv`、`.m4v`

传统 RAG 使用硅基流动嵌入和重排模型，内置批量嵌入、批次间隔和 429 限流重试。系统内部仍兼容历史 RAG-Anything 数据目录和索引结构，用于旧数据平滑迁移与读取，但该兼容能力不作为用户操作选项暴露。

### 2. 向量数据库管理

#### 创建新的知识库
通过 API 创建：首次导入时传入新的 `database` 名称即可自动登记。

#### 切换数据库
在API调用中指定 `database` 参数即可切换不同的数据库

### 3. 自定义技能开发

#### 创建新技能
1. 在项目根目录创建新的技能目录，名称必须以 `-skill` 结尾
2. 创建 `SKILL.md` 文件，定义技能配置
3. 实现技能的业务逻辑

**技能目录结构**:
```
new-skill/
├── SKILL.md          # 技能配置文件
├── assets/           # 资源文件
├── references/       # 参考文档
└── scripts/          # 脚本文件
```

## 故障排除

### 1. 系统无法响应

**症状**: 前端界面显示"连接失败"

**解决方案**:
- 检查RAG服务是否启动（端口8003）
- 检查AI陪练服务是否启动（端口8002）
- 检查防火墙设置，确保端口开放

### 2. 知识库检索失败

**症状**: 搜索结果为空或不准确

**解决方案**:
- 检查文件是否正确上传
- 确认查询关键词是否准确
- 检查向量数据库状态
- 如果文件状态为失败，点击文档列表中的「重试」
- 如果日志提示 429 TPM 限流，稍后重试或在 `.env` 中降低 `EMBEDDING_BATCH_SIZE`、增大 `EMBEDDING_BATCH_INTERVAL`
- 如果历史 RAG-Anything 数据读取异常，重启 RAG API 服务并检查 `storage/raganything/` 与 `storage/databases.json` 是否一致

### 3. 技能调用失败

**症状**: 技能无法正常启动或执行

**解决方案**:
- 检查技能目录名称是否以 `-skill` 结尾
- 确认 `SKILL.md` 文件格式正确
- 检查技能脚本是否有语法错误

## 性能优化

### 1. 提高检索速度

- 使用更高效的向量模型
- 定期清理过期数据
- 优化查询参数（如limit值）

### 2. 减少内存使用

- 关闭未使用的数据库
- 限制同时处理的文件数量
- 使用适当的批处理大小

### 3. 提升响应速度

- 使用GPU加速（如果可用）
- 优化模型加载策略
- 调整系统缓存设置

## 数据安全

### 1. 敏感信息保护

- 不要在配置文件中存储真实的API密钥
- 使用环境变量管理敏感信息
- 定期更新API密钥和访问凭证

### 2. 数据备份

- 定期备份向量数据库
- 备份重要的会话记录
- 建立数据恢复机制

### 3. 访问控制

- 限制API访问权限
- 使用HTTPS加密传输
- 实施适当的认证机制
