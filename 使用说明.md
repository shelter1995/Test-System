# 智学工作台 使用说明

> AI 销售话术陪练与 RAG 知识库系统 · 终端用户使用手册

本手册介绍智学工作台的**六大功能模块**、**典型工作流**和**常见问题 FAQ**。

如果软件还没安装，请先看 [安装说明.md](安装说明.md)。

---

## 1. 快速开始

只需 4 步就能上手：

1. **配置模型**（首次启动必做）：左下角 → **模型设置** → 填入 LLM、Embedding、Rerank 的 API Key，程序预设LLM服务商是MINMAX，嵌入和重排服务商是硅基流动，兼容OpenAI格式模型接入，可自行更改
2. **建知识库**：左侧 → **知识库** → 新建 → 上传产品文档/话术/案例
3. **练对练**：左侧 → **陪练系统** → 选场景 + 选知识库 → 开始对话
4. **看报告**：陪练结束后自动生成评分报告，也可去 **历史产物** 下载

## 2. 功能模块

### 2.1 总览

> 路径：左侧 → 📊 总览

进入后的"仪表盘"页面，显示：

- 服务运行状态（绿色=正常，红色=异常）
- 已建知识库数量、文件总数
- 近期生成的产物（陪练报告、解决方案）
- 增强解析组件状态（ffmpeg/Whisper/MinerU 是否就绪）

> 💡 启动异常时第一反应：来这里看"服务运行状态"是不是红的。

### 2.2 知识库

> 路径：左侧 → 📚 知识库

把产品资料、话术文档、案例 PDF 喂给 AI，让陪练和问答"有据可依"。

**操作流程**：
1. 点击"**新建知识库**" → 输入名称（如"商务彩铃产品手册"）→ 确定
2. 在知识库列表点击进入 → "**上传文件**"
3. 支持格式（基础）：
   - 文本：`.txt`、`.md`、`.csv`
   - PDF（可复制文字的）：`.pdf`
   - Office：`.docx`、`.xlsx`、`.pptx`
4. 支持格式（需先安装增强解析组件）：
   - 扫描版 PDF（图片型）
   - 老格式 Office：`.doc`、`.xls`、`.ppt`
   - 图片：`.png`、`.jpg`、`.jpeg`、`.bmp`、`.tiff`、`.webp`
   - 音频：`.mp3`、`.wav`、`.flac`、`.aac`、`.ogg`、`.m4a`、`.wma`
   - 视频：`.mp4`、`.mov`、`.mkv`、`.avi`、`.wmv`、`.webm`、`.m4v`
5. 上传后状态会从 `uploaded` → `rag_ingest` → `done` / `error`
6. 单个文件失败可点击"**重试**"，会换一种解析策略重试

**小贴士**：
- 每个文件 **≤ 200 MB** 解析更稳定
- 一个知识库建议 **≤ 500 个文件**，超过会拖慢检索
- 不要把多个产品的资料混在一个知识库

### 2.3 知识库问答

> 路径：左侧 → 💬 知识库问答

选中一个知识库，**基于该知识库内容**自由提问。

- 顶部下拉切换知识库
- 多轮对话，上下文会延续
- 每个回答底部显示**来源依据**（来自哪个文件哪一段）
- 没有找到答案时会明确说"未在知识库中检索到相关信息"

### 2.4 内容生成

> 路径：左侧 → ✏️ 内容生成

基于知识库自动生成结构化文档：

- **解决方案**（SCQA + MECE 架构）
- **培训讲义**（含目标、大纲、教学方法、评估）
- **测试题**（单选/多选/判断，含答案和解析）
- **README**（产品/项目说明）

生成的产物会出现在 **历史产物** 页，可下载为 Markdown。

### 2.5 陪练系统

> 路径：左侧 → 🎯 陪练系统

核心功能。模拟真实客户对话，训练销售话术。

**开始一次陪练**：
1. 顶部选择**知识库**（AI 会基于该知识库回答）
2. 选择**客户场景**（价格敏感型、技术挑剔型、决策谨慎型、竞品对比型等）
3. 选择**对应产品**
4. 点击"**开始对话**"

**对话中**：
- 你扮演销售，AI 扮演客户

- 每轮 AI 回复后，右侧以及对话框中实时显示本轮评分（多个维度）

  **结束后**：

- 自动生成**训练报告**：总分、各维度得分、亮点、改进建议

- 报告可下载，也可去 **历史产物** 查看

### 2.6 模型设置

> 路径：左侧 → ⚙️ 模型设置

首次启动**必须**配置，否则所有 AI 功能不可用。

需要填的项：
- **推理模型**（LLM）：用于对话生成、评分、内容生成
- **嵌入模型**（Embedding）：把文本转成向量
- **重排模型**（Rerank）：检索结果精排

每个模型填：服务商、API Key、模型名。填完点"测试连接"验证。

### 2.7 历史产物

> 路径：左侧 → 📁 历史产物

查看和下载历史内容生成产物

## 3. 典型工作流

### 3.1 销售新人入职

```
第 1 天：安装 → 装增强解析组件 → 配置模型
         ↓
第 2 天：建"产品手册"知识库，上传资料
         ↓
第 3 天：选"价格敏感型"场景，开始 5 轮陪练
         ↓
第 4 天：看报告，针对低分维度再去"知识库问答"复习
         ↓
第 5 天：再练一次 → 复盘改进
```

### 3.2 给客户出方案

```
1. 建"客户资料"知识库，上传客户背景材料
2. 用"知识库问答"摸清客户痛点
3. 切到"内容生成" → 选知识库 → 选"解决方案"
4. 填客户名称 + 需求 → 生成
5. 下载 Markdown → 二次加工后发给客户
```

## 4. 常见问题 FAQ

### Q1：上传视频/音频后报"Whisper transcription returned empty text."

**原因**：视频/音频**没有可识别的语音**。常见场景：
- 录屏视频只有 UI 操作画面，没有麦克风旁白
- 视频音轨完全静音或只有背景音乐
- 录音文件是环境噪音，没有说话

**解决办法**：
1. **双击播放**原视频/音频，确认是否有说话
2. 如果是录屏没有旁白 → 这种视频无法转写文本，属正常情况，可换有语音的版本
3. 如果有语音但识别不到：
   - 检查音量是否过低（剪辑软件可调大音量后重传）
   - 检查文件是否损坏（用其他播放器能正常播放）
4. 如果是**所有**视频都这样 → 可能是 Whisper 模型没下载完整，重装增强解析组件

### Q2：上传视频/音频报"Whisper transcription failed: [WinError 2]"

**原因**：ffmpeg/Whisper 没有正确安装。`imageio-ffmpeg` 包提供的 ffmpeg 二进制文件名带版本号（不是 `ffmpeg.exe`），依赖 PATH 搜索时会找不到。

**解决办法**：
1. **重装增强解析组件**：右上角菜单 → 安装/修复增强解析组件 → 开始安装
2. 安装完成后重启智学工作台
3. 如果仍报错 → 打开日志目录，把 `rag-service.log` 末尾 50 行发给开发人员

### Q3：上传扫描版 PDF 或图片报"未找到可用解析器"

**原因**：增强解析组件没装。

**解决办法**：右上角菜单 → 安装/修复增强解析组件 → 完成后再上传。

### Q4：上传文件后状态一直停在"rag_ingest"不变成"done"

**原因**：解析时间较长（PDF/视频可能 1–10 分钟）。

**排查步骤**：
1. 等 5 分钟以上，刷新页面看是否变化
2. 看 **总览** 页"增强解析组件状态"是否都是 ✅
3. 看 `rag-service.log` 日志最后几行（看是否在处理、还是卡住）
4. 长时间无变化 → 点击文件旁的"**重试**"

### Q5：主程序启动失败/打不开

**最常见原因**：**端口 8002 / 8003 被占用**。

**排查步骤**：
1. 打开"任务管理器" → 详细信息
2. 看是否有 `python.exe` 在运行且占用很多内存（残留的旧进程）
3. 结束所有 `python.exe` 进程后重启智学工作台
4. 如果还失败，打开命令行：
   ```cmd
   netstat -ano | findstr :8002
   netstat -ano | findstr :8003
   ```
   找到占用进程 PID，在任务管理器结束它
5. 极端情况：之前安装的智学工作台没卸载干净，按 [安装说明 → 卸载](安装说明.md#7-卸载) 重装

### Q6：双击图标后弹出"WebView2 Runtime 未安装"

**原因**：系统缺少 WebView2，且首次安装时网络异常没自动装上。

**解决办法**：
1. 让杀毒软件放行 Microsoft 下载域名 `go.microsoft.com`
2. 重新运行 EXE 安装包（会自动重试安装 WebView2）
3. 手动安装：访问 https://developer.microsoft.com/microsoft-edge/webview2/ 下载"Evergreen Standalone Installer"

### Q7：AI 答非所问 / 答案质量差

**排查步骤**：
1. **先看知识库**有没有相关资料 — 没资料 AI 只能瞎编
2. **检查 Embedding 模型**是否填写正确 — 错填会导致检索不到内容
3. **看 LLM 模型**是不是太弱的（如 `nano`、`haiku`）— 弱模型理解能力差
4. **换种问法**重试 — 知识库问答支持多轮，可追加"请引用原文"
5. 如果是陪练场景，检查**评分依据**看 AI 是不是按错的标准打分

### Q8：知识库问答返回"未在知识库中检索到相关信息"

**可能原因**：
1. 知识库是空的（没上传文件）
2. 文件还在解析中（状态不是 `done`）
3. 问的问题**不在上传的文档里**
4. Embedding 模型不匹配（中途换过 API Key 需重建向量库）

**解决办法**：
1. 在 **知识库** 页确认文件状态都是 `done`
2. 把问句拆短、用文档里的原话重试
3. 如果刚换过 Embedding 模型 → 删知识库重建（旧的向量已失效）

### Q9：安装增强解析组件卡在"下载模型"不动

**可能原因**：
1. 网络不通或被防火墙拦截

2. 模型源（ModelScope/Hugging Face）连接失败

3. 磁盘空间不足

  **解决办法**：

4. 关闭杀软/代理后重试

5. 检查数据目录所在磁盘剩余空间（至少 5 GB）

4. 仍失败 → 打开 `mineru-installer.log` 看具体卡哪一步

### Q10：对话很慢 / 一直转圈

**可能原因**：
1. LLM API 调用慢（服务商限流/网络问题）
2. 知识库太大检索慢
3. 同时开了多个对话标签页

**解决办法**：
1. 在 **总览** 页看"服务运行状态"是否正常
2. 切换到更小的 LLM 模型（牺牲质量换速度）
3. 大知识库（>1000 文件）会显著变慢，建议拆分

### Q11：升级后启动报"配置文件读取失败"

**原因**：升级过程中旧数据目录被破坏（多见于杀软误删文件）。

**解决办法**：
1. 从备份恢复 `数据目录\config\` 子目录
2. 没备份的话：重装智学工作台，**不删除数据目录**，程序会自动重建 config

### Q12：怎么把知识库迁移到另一台电脑？

见 [安装说明 → 数据目录迁移](安装说明.md#82-迁移到新电脑)。

## 5. 获取日志并反馈

当遇到 **FAQ 里没有覆盖的问题** 时，请按下面的步骤收集日志并发送给开发人员，能极大加快问题排查速度。

### 5.1 复现问题

确保你**至少复现过一次**这个 bug，并记下：
- 操作步骤（点了什么、上传了什么、什么时候开始报错）
- 报错信息原文（最好截图）
- 智学工作台版本（"关于"页面或安装包文件名）

### 5.2 打开日志目录

手动打开软件文件保存文件夹，默认在：
```
%LOCALAPPDATA%\智学工作台\Data\logs\
```
### 5.3 应该收集的日志文件

| 文件 | 何时需要 |
|------|---------|
| `desktop-host.log` | 主程序启动/崩溃/界面异常时 |
| `rag-service.log` | 知识库上传、解析、问答异常时 |
| `tutor-service.log` | 陪练对话、评分、报告异常时 |
| `mineru-installer.log` | 增强解析组件安装失败时 |

> 💡 体积较大的日志（>5 MB）会被自动轮转（保留 5 个历史文件），不需要清理。

### 5.4 收集步骤

1. **关闭智学工作台**（避免日志被截断）
2. 把相关日志文件复制到一个文件夹，例如 `智学工作台-日志-20260101\`
3. 如日志太大，**只保留报错前后各 200 行**：
   ```powershell
   Get-Content ".\rag-service.log" -Tail 200 > rag-末尾200行.log
   ```
4. 把这个文件夹打包成 zip，发给开发人员

### 5.5 反馈时附上的信息

为了帮开发人员快速定位，请同时提供：

- ✅ **问题描述**：在做什么、期望是什么、实际是什么
- ✅ **复现步骤**：1, 2, 3...
- ✅ **相关文件**：出错时上传/处理的文件名（不要发文件内容）
- ✅ **日志文件**：见 5.4
- ✅ **系统信息**：Windows 版本（`winver` 查看）、内存大小、是否用杀软
- ❌ **不要发**：API Key、知识库内容（可能含机密）

## 6. 术语表

- **RAG**：Retrieval-Augmented Generation，检索增强生成。AI 回答时先去知识库找资料再回答，避免瞎编。
- **Embedding**：把文本转成数学向量，让计算机能"理解"语义相似度。
- **Rerank**：检索结果的二次精排，挑出最相关的几条。
- **Whisper**：OpenAI 开源的语音转文字模型。
- **MinerU**：开源的 PDF/图片解析工具，专门处理扫描版和复杂版式。
- **ffmpeg**：音视频处理工具，用于抽音轨、转格式。
