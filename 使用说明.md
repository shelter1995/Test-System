# AI 话术陪练系统 — 使用说明

## 系统概述

企业 AI 工作台是一个集成知识库管理、AI 内容生成、话术陪练的 Web 应用。系统包含两个核心服务：

1. **RAG 知识库服务**（端口 8003）：负责文档解析、向量检索、图谱检索和混合检索
2. **AI 话术陪练系统**（端口 8002）：负责场景会话、MiniMax 对话生成、实时评估和内容生成

启动两个服务后，浏览器访问 **http://localhost:8002** 进入统一工作台。

## 快速开始

### 启动服务

```bash
# 终端 1：启动 RAG 核心服务
cd rag-anything-api
python start.py

# 终端 2：启动 AI 陪练系统
cd ai-tutor-system
python tutor_backend.py
```

### 打开工作台

浏览器访问：**http://localhost:8002**

工作台包含五个模块：

| 模块 | 功能 |
|------|------|
| **总览** | 查看服务状态、知识库数量、文件数量和最近产物 |
| **知识库** | 创建产品知识库，上传文件并导入，支持搜索测试 |
| **内容生成** | 选择知识库后生成解决方案、培训讲义、测试题和 README |
| **陪练系统** | 选择知识库、客户场景和产品后开始对话陪练 |
| **历史产物** | 查看并下载生成的 Markdown 文件 |

## 知识库管理

### 创建知识库

1. 进入工作台「知识库」页面
2. 点击「新建知识库」，输入名称（如"产品文档"）
3. 拖拽或选择文件上传（支持 PDF、Word、Excel、PPT、图片、音视频等）
4. 等待解析完成（进度条实时显示 SSE 进度）
5. 搜索框输入关键词测试检索

### 支持的文件格式

`.pdf` `.doc` `.docx` `.ppt` `.pptx` `.xls` `.xlsx` `.txt` `.md` `.jpg` `.png` `.bmp` `.mp4` `.mp3` `.wav`

### API 接口（供开发者参考）

#### 文件上传
```bash
curl -X POST "http://localhost:8003/ingest/upload" \
  -F "database=我的知识库" \
  -F "files=@/path/to/document.pdf"
```

#### 文本导入
```bash
curl -X POST "http://localhost:8003/ingest/text" \
  -H "Content-Type: application/json" \
  -d '{"text": "文本内容", "database": "我的知识库"}'
```

#### 目录批量导入
```bash
curl -X POST "http://localhost:8003/ingest/path" \
  -H "Content-Type: application/json" \
  -d '{"path": "/path/to/documents", "database": "我的知识库", "recursive": true}'
```

#### 文档搜索
```bash
curl -X POST "http://localhost:8003/search" \
  -H "Content-Type: application/json" \
  -d '{"query": "查询内容", "database": "我的知识库", "n_results": 5}'
```

#### 轻量上下文检索（陪练内部使用）
```bash
curl -X POST "http://localhost:8003/context" \
  -H "Content-Type: application/json" \
  -d '{"query": "查询内容", "database": "我的知识库", "n_results": 5}'
```

#### 系统状态
```bash
curl -s http://localhost:8003/status
```

## AI 话术陪练

### 使用流程

1. **选择场景**：进入「陪练系统」页面，从下拉菜单中选择角色扮演场景
2. **填写信息**：输入客户单位、产品名称、对话背景
3. **开始对话**：点击「开始陪练」，AI 扮演客户，你扮演销售代表
4. **实时反馈**：每轮对话结束后自动评分，右侧面板显示累计评分趋势
5. **结束报告**：点击「结束会话」生成详细评估报告

### 预设场景

| 场景 | 客户特征 | 应对策略 |
|------|---------|---------|
| **价格敏感型客户** | 关注预算和成本，会对比竞品价格 | 突出产品独特价值，提供 ROI 计算 |
| **技术挑剔型客户** | 关注技术参数和架构，质疑稳定性 | 提供技术细节，安排技术交流 |
| **决策谨慎型客户** | 多方比较，关注风险和服务保障 | 提供案例证明，强调服务保障 |
| **竞品对比型客户** | 在多家产品间比较 | 突出差异化优势 |

### 评分维度

系统从五个维度对每轮对话进行实时评分：

| 维度 | 权重 | 说明 |
|------|------|------|
| 开场话术 | 15% | 开场是否自然，是否建立信任 |
| 需求挖掘 | 20% | 是否了解客户真实需求和痛点 |
| 产品介绍 | 20% | 产品介绍是否匹配客户需求 |
| 异议处理 | 25% | 是否有效应对客户质疑和异议 |
| 促成技巧 | 20% | 是否推进成交或达成下一步 |

### 自定义场景

编辑 `ai-tutor-system/tutor_config.py` 中的 `DEFAULT_SCENARIOS`，添加新的场景配置：

```python
"new_scenario": {
    "id": "new_scenario",
    "name": "新场景名称",
    "ai_role": "客户角色",
    "user_role": "销售代表",
    "description": "场景描述",
    "customer_traits": ["特征1", "特征2"],
    "ai_strategy": ["策略1", "策略2"],
    "success_criteria": ["成功标准1"]
}
```

### 会话管理

- 会话自动保存到 `ai-tutor-system/tutor_data/sessions/` 目录
- 历史记录可在「陪练系统」右侧历史抽屉中查看
- 历史抽屉支持搜索、场景筛选和评分排序

## 内容生成

### 解决方案生成

进入「内容生成」页面，选择"解决方案"：

1. 选择目标知识库
2. 填写客户单位、决策人职位、客情关系
3. 输入客户痛点（最多 3 条）
4. 选择决策偏好
5. 点击生成，系统基于 RAG 检索和 SCQA+MECE 架构生成结构化方案

生成产物保存在 `generation_output/` 目录。

### 培训材料生成

进入「内容生成」页面，选择"培训材料"：

1. 选择目标知识库
2. 输入培训主题、对象、时长、目标
3. 选择题型及数量（单选/多选/判断/简答）
4. 设置难度分布（基础/进阶/挑战）
5. 点击生成，系统生成：培训讲义 + 测试题 + README

生成产物保存在 `generation_output/` 目录。

### 历史产物

进入「历史产物」页面，可查看和下载所有生成的 Markdown 文件。

## 故障排除

### 系统无法响应

**症状**：前端界面显示"连接失败"

**解决方案**：
- 检查 RAG 服务是否启动（端口 8003）
- 检查 AI 陪练服务是否启动（端口 8002）
- 检查防火墙设置，确保端口开放

### 知识库检索失败

**症状**：搜索结果为空或不准确

**解决方案**：
- 检查文件是否正确上传并完成解析
- 确认查询关键词是否准确
- 检查向量数据库状态：`curl -s http://localhost:8003/status`

### 内容生成失败

**症状**：点击生成后长时间无响应或报错

**解决方案**：
- 检查 MiniMax API Key 是否有效
- 确认已选择知识库且知识库中有内容
- 查看服务端日志排查具体错误

### 端口被占用

**症状**：启动时报 `Address already in use`

**解决方案**：
- 修改 `rag-anything-api/.env` 中的 `RAG_SERVICE_PORT`
- 修改 `ai-tutor-system/.env` 中的 `TUTOR_SERVICE_PORT` 和 `RAG_SERVICE_URL`
