# AI话术陪练系统 - 部署说明

## 项目概述

AI话术陪练系统是一个基于 MiniMax AI、传统 RAG 和 RAG-Anything 的本地角色扮演训练系统。系统包含两个核心服务：
- 知识库服务：默认使用传统 RAG 快速处理常见文档，也可切换 RAG-Anything 处理复杂 PDF、多模态和图谱增强场景。
- AI话术陪练系统：基于RAG知识库进行智能角色扮演训练。

## 环境要求

### 硬件要求
- CPU：双核以上处理器
- 内存：8GB以上（推荐16GB）
- 存储空间：10GB以上（包含模型文件和知识库数据）

### 软件要求
- Python 3.10 或更高版本
- Git（可选，用于版本控制）
- 现代浏览器（Chrome、Edge、Firefox等）
- MinerU（可选，仅 RAG-Anything 高级解析需要）

## 安装步骤

### 1. 解压分发包
```bash
# Windows系统
解压 ai-tutor-system_xxxxxx_xxxxxx.zip 到任意目录
```

### 2. 创建虚拟环境（推荐）
```bash
# Windows系统
cd ai-tutor-system
python -m venv venv
venv\Scripts\activate
```

### 3. 安装依赖包

#### 安装RAG核心服务依赖
```bash
cd rag-anything-api
pip install -r requirements.txt
```

#### 安装AI陪练系统依赖
```bash
cd ..\ai-tutor-system
pip install -r requirements_tutor.txt
```

### 4. 配置环境变量

#### RAG核心服务配置
```bash
cd rag-anything-api
copy .env.example .env
```
编辑 `.env` 文件：
```
# MiniMax AI配置
MINIMAX_API_KEY=your_api_key_here
MINIMAX_BASE_URL=https://api.minimaxi.com/v1

# 硅基流动配置
SILICONFLOW_API_KEY=your_api_key_here
SILICONFLOW_BASE_URL=https://api.siliconflow.cn
SILICONFLOW_MODEL=BAAI/bge-m3

# 服务配置
RAG_SERVICE_HOST=0.0.0.0
RAG_SERVICE_PORT=8003

# 默认知识库引擎
DEFAULT_RAG_ENGINE=traditional
```

#### AI陪练系统配置
```bash
cd ..\ai-tutor-system
copy .env.example .env
```
编辑 `.env` 文件：
```
# MiniMax AI配置
MINIMAX_API_KEY=your_api_key_here
MINIMAX_BASE_URL=https://api.minimaxi.com/v1
MINIMAX_MODEL=MiniMax-M2.7

# 服务配置
RAG_SERVICE_URL=http://localhost:8003
TUTOR_SERVICE_HOST=0.0.0.0
TUTOR_SERVICE_PORT=8002
```

## 启动服务

### 方法一：使用快速启动脚本（推荐）

```bash
start_services.bat
```

脚本会检查依赖、端口占用和服务健康状态，并自动打开 http://localhost:8002。

### 方法二：手动启动

#### 启动RAG核心服务
```bash
cd rag-anything-api
python start.py
```

#### 启动AI陪练系统
```bash
cd ai-tutor-system
uvicorn tutor_backend:app --host 0.0.0.0 --port 8002 --reload
```

## 验证部署成功

1. 访问 RAG服务API文档：http://localhost:8003/docs
2. 测试 `/status` 端点，确认服务正常运行
3. 打开统一工作台：http://localhost:8002
4. 在「模型设置」页面保存并测试推理、嵌入、重排模型
5. 在「知识库」页面上传 `.txt` 或 `.xlsx` 文件，确认导入完成
6. 在「知识库问答」页面切换知识库并测试问答和来源依据

## 常见问题处理

### 1. 端口被占用
- 错误信息: `Address already in use`
- 解决方案: 修改 `.env` 文件中的端口配置，或关闭占用端口的程序

### 2. 依赖安装失败
- 错误信息: `pip install` 失败
- 解决方案:
  ```bash
  # 更新pip
  python -m pip install --upgrade pip
  
  # 手动安装失败的包
  pip install package_name
  ```

### 3. 模型加载失败
- 错误信息: `Model not found` 或 `ImportError`
- 解决方案: 确认模型文件完整，检查模型路径配置

### 4. MiniMax AI API错误
- 错误信息: `API key invalid` 或 `Rate limit exceeded`
- 解决方案: 检查API密钥配置，确保余额充足

### 5. 硅基流动 429 限流
- 错误信息: `429 Too Many Requests` 或 `TPM limit reached`
- 解决方案: 降低 `EMBEDDING_BATCH_SIZE`，增大 `EMBEDDING_BATCH_INTERVAL`，或稍后点击失败文档「重试」

### 6. RAG-Anything PDF 重试失败
- 错误信息: `asyncio.locks.Lock ... is bound to a different event loop`
- 解决方案: 确认已重启 RAG API 服务加载最新代码，然后在文档列表点击「重试」

## 数据备份与迁移

### 备份知识库数据
```bash
# 备份知识库注册表
copy rag-anything-api\storage\databases.json your_backup_path

# 备份传统 RAG 存储
copy rag-anything-api\storage\traditional_rag your_backup_path

# 备份RAG-Anything存储
copy rag-anything-api\storage\raganything your_backup_path

# 备份解析输出
copy rag-anything-api\output your_backup_path
```

### 恢复数据
```bash
# 恢复注册表
copy your_backup_path\databases.json rag-anything-api\storage\

# 恢复存储
copy your_backup_path\raganything rag-anything-api\storage\

# 恢复解析输出
copy your_backup_path\output rag-anything-api\
```

## 部署到生产环境

### 配置防火墙
确保开放以下端口：
- 8003 (RAG-Anything知识库服务)
- 8002 (AI陪练系统)

### 使用进程管理器
```bash
# 使用pm2（需要先安装Node.js）
npm install -g pm2

# 启动RAG服务
cd rag-anything-api
pm2 start "python app.py" --name rag-service

# 启动陪练系统
cd ../ai-tutor-system
pm2 start "python tutor_backend.py" --name tutor-service
```

### 配置反向代理（Nginx示例）
```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    location /rag/ {
        proxy_pass http://localhost:8003/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    location /tutor/ {
        proxy_pass http://localhost:8002/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```
