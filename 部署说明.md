# AI话术陪练系统 - 部署说明

## 项目概述

AI话术陪练系统是一个基于MiniMax AI和全模态RAG技术的智能角色扮演训练系统。系统包含两个核心服务：
- RAG核心服务：提供文档、图片、音频、视频的智能检索功能
- AI话术陪练系统：基于RAG知识库进行智能角色扮演训练

## 环境要求

### 硬件要求
- CPU：双核以上处理器
- 内存：8GB以上（推荐16GB）
- 存储空间：10GB以上（包含模型文件和知识库数据）

### 软件要求
- Python 3.10 或更高版本
- Git（可选，用于版本控制）
- 现代浏览器（Chrome、Edge、Firefox等）

## 安装步骤

### 1. 解压分发包
```bash
# Windows系统
解压 ai-tutor-system_xxxxxx_xxxxxx.zip 到任意目录
```

### 2. 创建虚拟环境（推荐）
```bash
# Windows系统
cd ai-tutor-system
python -m venv venv
venv\Scripts\activate
```

### 3. 安装依赖包

#### 安装RAG核心服务依赖
```bash
cd rag-anything-api
pip install -r requirements.txt
```

#### 安装AI陪练系统依赖
```bash
cd ..\ai-tutor-system
pip install -r requirements_tutor.txt
```

### 4. 配置环境变量

#### RAG核心服务配置
```bash
cd rag-anything-api
copy .env.template .env
```
编辑 `.env` 文件：
```
# MiniMax AI配置
MINIMAX_API_KEY=your_api_key_here
MINIMAX_BASE_URL=https://api.minimax.chat/v1

# 系统配置
API_HOST=0.0.0.0
API_PORT=8003

# 模型配置
TEXT_EMBEDDING_MODEL=./models/sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2
IMAGE_EMBEDDING_MODEL=./models/clip/openai-clip-vit-base-patch32
```

#### AI陪练系统配置
```bash
cd ..\ai-tutor-system
copy .env.template .env
```
编辑 `.env` 文件：
```
# MiniMax AI配置
MINIMAX_API_KEY=your_api_key_here
MINIMAX_BASE_URL=https://api.minimax.chat/v1
MINIMAX_MODEL=abab6.5s-chat

# 服务配置
RAG_SERVICE_URL=http://localhost:8003
TUTOR_SERVICE_HOST=0.0.0.0
TUTOR_SERVICE_PORT=8002
```

## 启动服务

### 方法一：使用快速启动脚本（推荐）

#### 启动RAG核心服务
```bash
cd rag-anything-api
python start.py
```

#### 启动AI陪练系统
在另一个命令窗口中：
```bash
cd ai-tutor-system
start_tutor.bat
```

### 方法二：手动启动

#### 启动RAG核心服务
```bash
cd rag-anything-api
python start.py
```

#### 启动AI陪练系统
```bash
cd ai-tutor-system
uvicorn tutor_backend:app --host 0.0.0.0 --port 8002 --reload
```

## 验证部署成功

1. 访问 RAG服务API文档：http://localhost:8003/docs
2. 测试 `/status` 端点，确认服务正常运行
3. 打开AI陪练系统前端界面，测试聊天功能
4. 测试知识库检索功能，确认能正常搜索文档

## 常见问题处理

### 1. 端口被占用
- 错误信息: `Address already in use`
- 解决方案: 修改 `.env` 文件中的端口配置，或关闭占用端口的程序

### 2. 依赖安装失败
- 错误信息: `pip install` 失败
- 解决方案:
  ```bash
  # 更新pip
  python -m pip install --upgrade pip
  
  # 手动安装失败的包
  pip install package_name
  ```

### 3. 模型加载失败
- 错误信息: `Model not found` 或 `ImportError`
- 解决方案: 确认模型文件完整，检查模型路径配置

### 4. MiniMax AI API错误
- 错误信息: `API key invalid` 或 `Rate limit exceeded`
- 解决方案: 检查API密钥配置，确保余额充足

## 数据备份与迁移

### 备份知识库数据
```bash
# 备份知识库注册表
copy rag-anything-api\storage\databases.json your_backup_path

# 备份RAG-Anything存储
copy rag-anything-api\storage\raganything your_backup_path

# 备份解析输出
copy rag-anything-api\output your_backup_path
```

### 恢复数据
```bash
# 恢复注册表
copy your_backup_path\databases.json rag-anything-api\storage\

# 恢复存储
copy your_backup_path\raganything rag-anything-api\storage\

# 恢复解析输出
copy your_backup_path\output rag-anything-api\
```

## 部署到生产环境

### 配置防火墙
确保开放以下端口：
- 8003 (RAG-Anything知识库服务)
- 8002 (AI陪练系统)

### 使用进程管理器
```bash
# 使用pm2（需要先安装Node.js）
npm install -g pm2

# 启动RAG服务
cd rag-anything-api
pm2 start "python app.py" --name rag-service

# 启动陪练系统
cd ../ai-tutor-system
pm2 start "python tutor_backend.py" --name tutor-service
```

### 配置反向代理（Nginx示例）
```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    location /rag/ {
        proxy_pass http://localhost:8003/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    location /tutor/ {
        proxy_pass http://localhost:8002/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```
