# AI 话术陪练系统 - 部署说明

## 项目概述

Test-System 是一个本地双服务系统：

- 知识库服务：`rag-anything-api/`，FastAPI，默认端口 `8003`。用户可见能力统一为传统 RAG（向量检索 + 重排），内部仅保留历史 RAG-Anything 数据兼容读取能力。
- AI 话术陪练系统：`ai-tutor-system/`，FastAPI + 原生 JS 前端，默认端口 `8002`。负责统一工作台、陪练会话、评分、内容生成和模型设置。

## 环境要求

- Python 3.12-3.13
- Git
- 现代浏览器（Chrome、Edge、Firefox 等）
- 至少 10 GB 可用磁盘空间
- 可用的 MiniMax API Key 和 SiliconFlow API Key

可选系统依赖：

- LibreOffice：解析 `.doc`、`.xls`、`.ppt` 旧 Office 文件
- ffmpeg：从视频中抽取音轨
- MinerU CLI：扫描 PDF、复杂 PDF、图片 OCR
- openai-whisper：音频和视频语音转写，随 RAG requirements 安装

## 从 clone 到启动

```powershell
git clone <Test-System-仓库地址>
cd Test-System

python -m venv .venv
.\.venv\Scripts\activate
python -m pip install --upgrade pip

pip install -r rag-anything-api\requirements.txt
pip install -r ai-tutor-system\requirements_tutor.txt

copy rag-anything-api\.env.example rag-anything-api\.env
copy ai-tutor-system\.env.example ai-tutor-system\.env
```

编辑两个 `.env` 文件，至少填入：

```ini
# rag-anything-api/.env
MINIMAX_API_KEY=your-minimax-key
SILICONFLOW_API_KEY=your-siliconflow-key
ENABLE_RERANK=true
DEFAULT_RAG_ENGINE=traditional

# ai-tutor-system/.env
MINIMAX_API_KEY=your-minimax-key
RAG_SERVICE_URL=http://localhost:8003
```

启动：

```bat
start_services.bat
```

脚本会检查服务健康状态，必要时分别启动 RAG 服务和 Tutor 服务，然后打开 `http://localhost:8002`。

## 手动启动

终端 1：

```powershell
.\.venv\Scripts\activate
cd rag-anything-api
python start.py
```

终端 2：

```powershell
.\.venv\Scripts\activate
cd ai-tutor-system
python tutor_backend.py
```

访问地址：

- 统一工作台：http://localhost:8002
- Tutor API 文档：http://localhost:8002/docs
- RAG API 文档：http://localhost:8003/docs
- RAG 健康检查：http://localhost:8003/health

## 验证部署

1. 打开 `http://localhost:8002`，确认总览页面能显示服务状态。
2. 打开「模型设置」，保存并测试推理模型、嵌入模型和重排模型。
3. 打开「知识库」，新建知识库并上传 `.txt`、`.pdf`、`.docx` 或 `.xlsx` 文件。
4. 打开「知识库问答」，选择知识库后测试问答和来源依据。
5. 打开「陪练系统」，选择知识库和场景后开始对话。

命令行检查：

```powershell
curl http://localhost:8003/health
curl http://localhost:8003/status
curl http://localhost:8002/api/status
```

## 支持的导入格式

- 文档：`.pdf`、`.doc`、`.docx`、`.xls`、`.xlsx`、`.ppt`、`.pptx`、`.txt`、`.md`、`.csv`
- 图片：`.png`、`.jpg`、`.jpeg`、`.bmp`、`.tiff`、`.tif`、`.webp`
- 音频：`.mp3`、`.wav`、`.flac`、`.aac`、`.ogg`、`.m4a`、`.wma`
- 视频：`.mp4`、`.avi`、`.mkv`、`.mov`、`.webm`、`.wmv`、`.m4v`

解析链路会自动选择解析器，不需要用户手动切换引擎。

## 常见问题

### 端口被占用

修改 `.env`：

```ini
# rag-anything-api/.env
RAG_SERVICE_PORT=8004

# ai-tutor-system/.env
RAG_SERVICE_URL=http://localhost:8004
```

如需修改 Tutor 端口：

```ini
TUTOR_SERVICE_PORT=8005
```

### 依赖安装失败

```powershell
python -m pip install --upgrade pip setuptools wheel
pip install -r rag-anything-api\requirements.txt
pip install -r ai-tutor-system\requirements_tutor.txt
```

如果 MinerU 或 Whisper 下载模型较慢，可以先只验证 `.txt`、`.md`、`.csv`、`.docx`、`.xlsx` 等轻量格式，之后再补齐 OCR/转写能力。

### 旧 Office 文件解析失败

安装 LibreOffice，并确保 `soffice` 在 `PATH` 中。也可以在 `rag-anything-api/.env` 中配置：

```ini
LIBREOFFICE_PATH=C:\Program Files\LibreOffice\program\soffice.exe
```

### 视频或音频解析失败

确认 `ffmpeg` 可执行，并且 Whisper 已安装。视频会先通过 ffmpeg 抽取为 16 kHz 单声道音频，再交给 Whisper 转写。

### 上传后提示 429 Too Many Requests

这是嵌入模型侧的 TPM 限流。降低 `EMBEDDING_BATCH_SIZE`，增大 `EMBEDDING_BATCH_INTERVAL`，或等待限流窗口恢复后在文档列表点击「重试」。

## 数据备份

建议备份：

- 知识库注册表：`rag-anything-api\storage\databases.json`
- 传统 RAG 数据：`rag-anything-api\storage\traditional_rag\`
- 历史兼容数据：`rag-anything-api\storage\raganything\`
- 模型设置：`rag-anything-api\storage\model_settings.json`
- 陪练会话：`ai-tutor-system\tutor_data\`
- 内容生成产物：`generation_output\`
