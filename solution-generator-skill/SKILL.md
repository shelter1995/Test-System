## 🚨 刚性约束（必须遵守）

### 数据获取约束

**禁止**：
- ❌ 编写 Python 脚本获取数据
- ❌ 使用 database_manager.py 或其他 Python 代码
- ❌ 任何形式的编程方式访问数据库
- ❌ 虚构内容或编造信息

**必须**：
- ✅ 使用 Invoke-WebRequest 调用 API
- ✅ 通过 HTTP API 端点获取所有数据
- ✅ 每条信息必须标注源文件名称
- ✅ 基于实际检索结果生成内容

**可用的 API 端点**：

```powershell
# 获取数据库列表
Invoke-WebRequest -Uri "http://localhost:8003/db/list" | ConvertFrom-Json | ConvertTo-Json -Depth 3

# 获取数据库统计信息
Invoke-WebRequest -Uri "http://localhost:8003/db/stats/{database_name}" | ConvertFrom-Json | ConvertTo-Json -Depth 3

# 搜索知识库（语义检索）
$body = @{
    query = "搜索关键词"
    n_results = 10
    database = "{database_name}"
} | ConvertTo-Json
Invoke-WebRequest -Uri "http://localhost:8003/search" -Method POST -Body $body -ContentType "application/json" | ConvertFrom-Json | ConvertTo-Json -Depth 5
```

**重要提醒**：
- 每次获取数据都必须通过 API
- API 调用后使用 ConvertTo-Json 格式化输出
- 绝不编写 Python 脚本处理数据

### 内容生成约束

**禁止**：
- ❌ 生成提纲式、目录式的内容
- ❌ 只列出要点而不展开说明
- ❌ 使用"..."、省略号或"待补充"等占位符
- ❌ 生成需要讲师额外发挥的骨架内容
- ❌ 内容过于简略，无法独立使用

**必须**：
- ✅ **每个章节都要有完整的叙述内容**
- ✅ **详细展开每个要点，提供完整的讲解内容**
- ✅ **包含具体的案例描述、数据分析、实操步骤**
- ✅ **提供可直接使用的话术模板、工具清单**
- ✅ **内容要能够独立成篇，直接用于商务沟通，无需额外补充**
- ✅ **每个知识点都要有充分的说明和举例**
- ✅ **案例要完整，包含背景、过程、结果、关键话术**
- ✅ **话术要完整，提供具体可用的对话示例**

**内容深度要求**：
- 理论讲解：不仅要"是什么"，更要"为什么"、"怎么做"
- 案例分析：完整的案例故事，不是简单概括
- 话术模板：完整的对话脚本，不是要点提示
- 操作步骤：逐步详细的执行指南，不是简单列表
